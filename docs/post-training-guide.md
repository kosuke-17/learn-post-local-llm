# ポストトレーニング技術解説

## 全体像

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ ベースモデル  │ → │  LoRA       │ → │  GGUF変換    │ → │  Ollama     │
│ (Qwen2.5-7B)│    │ fine-tuning │    │  (量子化)    │    │ (ローカル実行)│
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
   事前学習済み        追加学習          フォーマット変換      推論サーバー
   汎用モデル         ドメイン特化        軽量化              デプロイ
```

## 1. ベースモデル: Qwen2.5-7B-Instruct

### なぜ既存モデルを使うのか

LLM をゼロから学習するには膨大なデータ（数兆トークン）と計算資源（数千 GPU × 数ヶ月）が必要。ポストトレーニングでは、既に汎用的な言語能力を獲得済みのモデルをベースに、少量のデータで目的の振る舞いを追加する。

### モデルの選定理由

| モデル | パラメータ数 | 特徴 |
|--------|------------|------|
| Llama 3.2 3B | 30億 | 軽量で高速。実験向き |
| **Qwen2.5-7B** | **70億** | **日本語性能が高い。本プロジェクトで採用** |
| Llama 3.1 8B | 80億 | 英語中心。日本語はやや弱い |

Qwen2.5 は中国 Alibaba が開発したモデルで、日本語を含む多言語に強い。7B（70億パラメータ）は Apple Silicon 32GB メモリで動作可能なサイズ。

### 4bit 量子化

`Qwen2.5-7B-Instruct-4bit` の `4bit` は量子化を意味する。通常 1 パラメータあたり 16bit（2バイト）のところを 4bit（0.5バイト）に圧縮し、メモリ使用量を約 1/4 に削減する。精度は若干落ちるが、消費者向けハードウェアで動かすには必須の技術。

```
FP16 (非量子化): 7B × 2バイト = 約14GB
4bit (量子化):   7B × 0.5バイト = 約3.5GB  ← Macで余裕で動く
```

## 2. LoRA (Low-Rank Adaptation)

### LoRA とは

モデル全体の重みを更新するのではなく、各層に小さな「アダプター」行列を挿入して、そこだけを学習する手法。

```
従来の fine-tuning:
  モデル全体（70億パラメータ）を更新 → メモリ・時間が膨大

LoRA:
  元のモデルは凍結（変更しない）
  小さなアダプター（数百万パラメータ）だけ学習 → 省メモリ・高速
```

### 仕組み

元の重み行列 W に対して、低ランク行列の積 A × B を加算する:

```
出力 = W × 入力 + (A × B) × 入力
        ↑ 凍結       ↑ 学習対象
```

A と B は元の行列よりずっと小さい（低ランク）ため、学習パラメータ数が大幅に減る。

### 本プロジェクトの設定

```
--lora-layers 16      # 16層にアダプターを挿入
--learning-rate 1e-5   # 学習率（小さめ = 元の能力を壊さない）
--iters 200            # 学習イテレーション数
--batch-size 4         # 一度に処理するデータ数
```

## 3. データセット

### 形式: ChatML

mlx-lm は ChatML 形式の JSONL を使う。1行が1つの会話。

```json
{
  "messages": [
    {"role": "user", "content": "ミミッキュの育成論を教えて"},
    {"role": "assistant", "content": "ミミッキュはばけのかわが強くて..."}
  ]
}
```

モデルはこの Q&A ペアから「ユーザーの質問に対してどう回答すべきか」を学習する。

### データ分割

| ファイル | 件数 | 用途 |
|---------|------|------|
| train.jsonl | 40件 | 学習に使用 |
| valid.jsonl | 5件 | 学習中の過学習チェック |
| test.jsonl | 5件 | 学習後の最終評価 |

## 4. MLX

Apple が開発した Apple Silicon 専用の機械学習フレームワーク。GPU（Metal）を直接利用して高速に推論・学習を行う。

PyTorch + CUDA（NVIDIA GPU 向け）の Apple Silicon 版と考えるとわかりやすい。

```
NVIDIA GPU の世界:  PyTorch + CUDA
Apple Silicon の世界: MLX + Metal   ← 本プロジェクト
```

`mlx-lm` は MLX 上で LLM の推論・fine-tuning・変換を行うためのライブラリ。

## 5. GGUF 変換

### なぜ変換が必要か

MLX 形式のモデルは MLX でしか動かない。Ollama で使うには GGUF（GPT-Generated Unified Format）形式に変換する必要がある。

```
MLX 形式 (.safetensors)  →  llama.cpp で変換  →  GGUF 形式 (.gguf)
```

### 量子化オプション

変換時に量子化レベルを指定できる:

| オプション | サイズ目安 (7B) | 品質 |
|-----------|---------------|------|
| f16 | 約14GB | 最高（非量子化） |
| q8_0 | 約7GB | 高品質 |
| q4_0 | 約3.5GB | 軽量、品質やや低下 |

本プロジェクトでは `q8_0` を採用。品質とサイズのバランスが良い。

## 6. Ollama

ローカルで LLM を動かすためのランタイム。Docker のように、モデルを `ollama run モデル名` で簡単に実行できる。

### Modelfile

Ollama にモデルを登録するための設定ファイル。Docker の Dockerfile に相当する。

```
FROM ./pokemon-trainer.gguf          # ベースとなるモデルファイル
SYSTEM あなたはポケモントレーナーだ...  # システムプロンプト
PARAMETER temperature 0.7            # 生成の多様性（高い = 多様、低い = 確定的）
PARAMETER num_ctx 4096               # コンテキスト長（入力+出力の最大トークン数）
TEMPLATE """..."""                    # プロンプトテンプレート（ChatML形式）
```

## 技術スタックまとめ

| レイヤー | 技術 | 役割 |
|---------|------|------|
| データ | JSONL (ChatML) | 学習データの形式 |
| ベースモデル | Qwen2.5-7B-Instruct-4bit | 事前学習済み汎用LLM |
| 学習フレームワーク | MLX + mlx-lm | Apple Silicon上でのfine-tuning |
| 学習手法 | LoRA | パラメータ効率的なfine-tuning |
| 変換 | llama.cpp | MLX → GGUF形式変換 |
| 推論ランタイム | Ollama | ローカルでのモデル実行 |
| パッケージ管理 | uv | Python依存関係の管理 |
